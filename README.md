# Branch de Continuidade do Projeto
---
A branch **`main`** contém exatamente a versão do projeto que foi desenvolvida e entregue dentro do prazo da atividade.
Esta branch (**`renan`**) foi criada para que eu possa continuar evoluindo o projeto como forma de estudo e aprimoramento pessoal. As alterações feitas aqui — como melhorias, refatorações e novas funcionalidades — não modificam a versão que foi oficialmente entregue.
---
Assim, a branch principal **`main`** permanece como registro fiel da entrega original, enquanto esta branch serve apenas para continuidade dos estudos.
---
### 📌 TAREFA EXTRA
Treinar o padrão **Conventional Commits**
---
## 📖 Formato Padrão
- `tipo` → categoria da mudança  
- `escopo` → parte do sistema afetada (opcional, mas recomendado)  
- `descrição` → frase curta, clara e no presente  
---
# 🏷️ Tipos de Commit
| Tipo      | Significado |
|-----------|------------|
| feat      | Nova funcionalidade |
| fix       | Correção de bug |
| docs      | Alterações na documentação |
| style     | Formatação/estilo (sem alterar lógica) |
| refactor  | Refatoração (sem alterar comportamento) |
| test      | Criação ou modificação de testes |
| chore     | Tarefas internas, configurações e manutenção |
| perf      | Melhoria de performance |
| build     | Alterações no sistema de build ou dependências |
| ci        | Mudanças relacionadas à integração contínua |
| revert    | Reversão de commit anterior |
---
#### 🧪 Exemplos Práticos
```
feat(posts): adiciona sistema de comentários
fix(auth): corrige erro de validação de senha
refactor(api): reorganiza estrutura das rotas
perf(database): otimiza consulta de listagem
build(project): atualiza dependências do npm
```
💡 Dica: sempre escreva a descrição no **presente**, de forma curta e objetiva.
